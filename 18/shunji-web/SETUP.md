# 順記燒臘館 · 設定教學

## ⚠️ 先講清楚：**完全不需要 LINE 官方帳號**

你用的是**自己平常在用的那個 LINE 帳號**。

- ✅ LINE Developers 後台 → 用你自己的 LINE 掃碼登入就好
- ✅ **免費**，不用申請官方帳號、不用營業登記、不用綁信用卡
- ✅ 你只是去 LINE 借一顆「登入按鈕」，讓客人用**他自己的 LINE** 登入你的網站

過程中如果看到 **`Linked LINE Official Account`** 或 **`Bot link feature`** 這種欄位，
**留空 / 選 None / 選 Off 就好**，那些才是官方帳號用的，你用不到。

---

## 你的問題：網頁能不能在手機上直接跑 LINE 登入認證？

**能。而且不用伺服器、不用花錢，你現在的 GitHub Pages 就能跑。**

客人在手機瀏覽器打開網頁，按「LINE 登入」，會**跳出 LINE 真正的授權畫面**：

```
        ┌──────────────────────────────┐
        │        順記燒臘館              │
        │                              │
        │   要求存取以下權限：            │
        │    ・你的顯示名稱與大頭貼        │
        │                              │
        │    [ 取消 ]      [ 允許 ]     │
        └──────────────────────────────┘
```

按「允許」→ 綁定完成，可以點餐。
按「取消」→ 進不來，點不了餐。**每個人都必須有 LINE、必須按允許。**

> 你原本的網站按了沒反應，是因為 `config.js` 裡 `var LIFF_ID = '';` **一直是空的**。
> **不是做不到，是還沒去 LINE 申請。**

# 設定（只有 2 步，約 5 分鐘）

## 步驟 1️⃣　去 LINE 申請

打開 **<https://developers.line.biz/console/>**（手機就能做）

### ① 用你自己的 LINE 登入

> 點 **【LINE 帳號登入】** → 掃碼 / 輸入帳密
> 第一次會叫你同意開發者條款，按同意。
>
> **就是你平常那個 LINE，不用另外申請什麼。**

### ② 建立 Provider

> **Create a new provider** → 名稱填 `順記燒臘館` → **Create**
>
> （這只是分類用的資料夾，建一次就好）

### ③ 建立 LINE Login 頻道

> **Create a new channel** → 選 **【LINE Login】**
>
> | 欄位 | 填什麼 |
> |---|---|
> | Channel name | `順記燒臘館` |
> | Channel description | `線上訂餐` |
> | **App types** | ☑ **Web app**　←　**一定要勾！** |
> | Email address | 你的 email |
>
> → **Create**

### ④ 加一個 LIFF app　★ 最重要

> 進到剛建好的頻道 → 點上排的 **【LIFF】** 頁籤 → **Add**
>
> | 欄位 | 填什麼 |
> |---|---|
> | LIFF app name | `順記點餐` |
> | Size | **Full** |
> | **Endpoint URL** | **`https://wsad952701-beep.github.io/123/13/`** |
> | **Scopes** | ☑ **profile**　☑ **openid**　←　**兩個都要勾！** |
> | Bot link feature | **Off**　←　這是官方帳號用的，關掉 |
>
> → **Add**

> ⚠️ **Endpoint URL 一定要一字不差**
> 就是你手機網址列看到的那一串，**結尾要有斜線 `/`**。

### ⑤ 複製 LIFF ID

加好後會出現一組 **LIFF ID**：

```
2001234567-AbCdEfGh
```

### ⑥ ★ 把頻道發佈（不做的話客人一律登入失敗）

> 頻道名稱旁邊有個 **`Developing`** 標籤 → 點它 → 改成 **`Published`**

---

## 步驟 2️⃣　把 LIFF ID 貼進 config.js

1. 手機打開 **github.com/wsad952701-beep/123** → 進 **`13`** 資料夾
2. 點 **`config.js`** → 右上角 **✏️**
3. 找到：

   ```js
   var LIFF_ID = '';
   ```

4. 改成：

   ```js
   var LIFF_ID = '2001234567-AbCdEfGh';
   ```

5. **Commit changes** → **Commit changes**

等 1～2 分鐘，**就好了。**

# 訂單怎麼到你手上？

## 說實話：訂單還是要貼到你的 LINE 社群

**為什麼不能自動送進社群？**

LINE 官方文件白紙黑字寫著：

> *「目前 LIFF 在 OpenChat（社群）中並未獲得官方支援，部分功能無法運作。」*

也就是說，**LINE 自己不讓網頁直接往社群發訊息**。任何人都做不到，不是我們的問題。

所以網站會**自動判斷**，用當下最順的方式：

| 客人的情況 | 網站會怎麼做 | 客人要做什麼 |
|---|---|---|
| **一般手機瀏覽器**<br>（最常見） | 自動複製訂單 → 一鍵開啟你的社群 | **長按輸入框 → 貼上 → 送出**<br>（約 10 秒） |
| 從 LINE **群組**開網頁 | 訂單直接發進群組 | 什麼都不用做 ✨ |
| 從 LINE **社群**開網頁 | 自動退回「複製 + 貼上」 | 長按 → 貼上 → 送出 |

**送出後的畫面會清楚寫著三個步驟：**

```
    ①            ②            ③
  點下面       長按          貼上
  進社群      輸入框         送出

  [ ① 點我開啟 LINE 社群 ]     ← 綠色大按鈕
  [ 我已經貼上了，清空點餐單 ]
  [ 再複製一次訂單 ]
```

> 💡 **點餐單不會提早清空** —— 一定要等客人按下「我已經貼上了」才清。
> 免得他還沒貼上，單子就不見了。

## 想更省事？（選配，2 個勾勾）

如果你有一個 **LINE 群組**（不是社群），或客人會**加你 LINE 好友**：

> LINE Developers → **LIFF** 頁籤 → 點你的 LIFF app →
> - Scopes 再加勾 **`chat_message.write`**
> - 把 **Share target picker** 打開

這樣客人按「送出訂單」會跳出「**要傳給誰**」的選單，選你 → 訂單直接傳過來，
**完全不用複製貼上**。

> ⚠️ 這個選單裡**只會出現好友和群組，不會出現社群**（LINE 的限制）。
> 不開也沒關係，會自動退回「複製 + 貼上」，一樣能訂。

# 出問題怎麼辦

| 客人說 | 原因 | 怎麼修 |
|---|---|---|
| **按 LINE 登入沒反應** | `config.js` 的 `LIFF_ID` 是空的 | 回步驟 2 |
| **「LINE 登入暫時無法使用」** | Endpoint URL 跟實際網址對不上 | 回步驟 1-④，一字一字比對，**結尾要有 `/`** |
| **一直登入失敗** | 頻道還停在 `Developing` | 回步驟 1-⑥ 改成 `Published` |
| **登入了但拿不到名字** | Scopes 沒勾 `profile` | 回步驟 1-④ 補勾 |
| **改了 config.js 沒變化** | GitHub / 瀏覽器快取 | 等 2 分鐘，關掉分頁重開 |
| **在 LINE 社群裡開網頁怪怪的** | LINE 不支援社群裡跑 LIFF | 叫客人**用瀏覽器開**（社群裡點連結 → 右上角「⋯」→ 用預設瀏覽器開啟） |

# 想改東西

### 改價格 / 加菜 / 下架

只要改 **`index.html`**。搜尋品名（例如「順記三寶飯」），
把 `data-price="140"` 和旁邊顯示的 `140` 一起改掉。

> **加餐點面板會自動同步**，不用改第二次。

### 改營業時間

`index.html` 搜尋 `var HOURS`。數字是「從午夜起算的分鐘數」：

```js
1: [[660,810],[960,1140]],   // 週一 11:00–13:30、16:00–19:00
6: [],                       // 週六公休
```
> 660 = 11×60 → 11:00　　1140 = 19×60 → 19:00

### 讓客人先挑菜、送出時才登入

`index.html` 搜尋 `LOGIN_BEFORE_ADD`，改成 `false`。

### 換社群連結

`config.js` 的 `COMMUNITY_URL`。

# 檔案怎麼放

把資料夾裡的東西全部丟進 GitHub 倉庫的 **`13`** 資料夾，覆蓋舊的：

```
123/13/
├── index.html      ← 網站
├── config.js       ← ★ 只要改這一個檔案
├── og-image.jpg
├── .nojekyll
└── assets/         ← 24 張照片（本來就有，可以不用重傳）
```

**其實你只要換 `index.html` 和 `config.js` 兩個檔案就好**，照片本來就在上面了。

**手機上的做法：**
1. `github.com/wsad952701-beep/123` → 進 `13` 資料夾
2. 右上角 **Add file → Upload files**
3. 丟進 `index.html` 和 `config.js`（會自動覆蓋）
4. **Commit changes**
